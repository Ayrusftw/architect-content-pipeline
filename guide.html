Overview
The Three
tools.

Every piece of content in this series was produced by three tools working in sequence. You don't need to be a developer to use them — you need to understand what each one does and when it hands off to the next.

01

ComfyUI Cloud

Generates the illustrated scene. Character, environment, mood. No local install.

02

Python PIL

Stamps the typography. Anton headline, Playfair emphasis, gray body. Pixel-precise.

03

Claude Code

Writes the scripts. You describe what you want. It writes the Python. You run it.

The flow:

Step Tool Input Output
1. Generate scene ComfyUI Cloud Text prompt + reference image Background PNG
2. Stamp text Python PIL Background PNG + script Final slide PNG
3. Write script Claude Code Your description Python script
4. Post Instagram Cover + slides Carousel post
Canvas size

All slides are 1856 × 2304 px — the 4:5 portrait format Instagram uses for carousels. Every script in this pipeline uses these dimensions.

Setup — Part 1 of 3
Python
the foundation.

Everything in this pipeline runs on Python 3.11. Not 3.10. Not 3.12. Version 3.11 specifically — this is the version ComfyUI Cloud and PIL play nicely with.

1

Check if Python 3.11 is installed

Open Terminal and run:

python3.11 --version

If you see Python 3.11.x — skip to Step 3. If not, continue to Step 2.

2

Install Python 3.11 via Homebrew

If you don't have Homebrew, install it first:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Then install Python 3.11:

brew install python@3.11
3

Install the required packages

Navigate to your project folder and run:

pip3.11 install -r requirements.txt

This installs: Pillow, requests, websockets, python-dotenv.

Why 3.11 specifically?

The ComfyUI Cloud WebSocket connection uses async features that behave differently across Python versions. 3.11 is the version this pipeline was built and tested on.

Setup — Part 2 of 3
ComfyUI
Cloud.

ComfyUI Cloud is the hosted version of ComfyUI — you generate images via API without installing anything locally. This pipeline uses the Nano Banana 2 model (Gemini Flash Image).

1

Create an account

Go to cloud.comfy.org and sign up. Free to create — you pay per generation.

2

Add credits

Go to Settings → Billing and add a small amount. Nano Banana 2 at 2K resolution costs roughly $0.02–0.05 per image. $5 covers a lot of testing.

3

Get your API key

Go to Settings → API Keys → Create new key. Copy it immediately — you only see it once.

4

Add it to .env

In your project folder, copy the example file:

cp .env.example .env

Open .env and paste your key:

COMFYUI_API_KEY=paste_your_key_here

This file is gitignored — your key never leaves your machine.

5

Test it

Add a photo of yourself as reference.jpg in the project folder, then run:

python3.11 generate-cover.py

Output saves to outputs/cover.png and opens automatically.

The authentication error

If you get "authentication method not allowed" — this is the most common first error. It means the API key isn't being passed in the request body. The scripts in this repo already handle this correctly via extra_data. Don't change that part.

Setup — Part 3 of 3
Claude
Code.

Claude Code is the AI coding assistant that wrote every script in this pipeline. You describe what you want in plain language. It writes the Python. You run it and give feedback.

You don't need to know how to code to use it. You need to know how to describe what you want.

1

Install Node.js

Claude Code requires Node.js 18 or higher. Check if you have it:

node --version

If not, download from nodejs.org and install.

2

Install Claude Code

npm install -g @anthropic-ai/claude-code
3

Get an Anthropic API key

Go to console.anthropic.com → sign up → API Keys → create new. A typical scripting session costs cents.

4

Start Claude Code

Navigate to your project folder, then run:

cd your-project-folder
claude

It will prompt for your API key the first time. After that it opens instantly.

5

Ask it to write a slide

Try this as your first prompt:

Add a make_03() function to generate-episode.py.
Use make_illustrated() with:
- Headline: "THREE MONTHS IN."
- Emphasis: "still learning."
- Body: "But building something real."

It writes the code. You run the script. You give feedback. That's the loop.

How I used it for this series

I described each slide in plain language. Claude Code wrote the PIL function. I ran it, opened the output, gave feedback ("push the text lower", "make the headline bigger"), and it revised. No manual Python editing for 9 episodes.

Using the pipeline
Your first
cover image.

The cover is always generated first. It sets the visual identity for the whole episode — character, scene, mood.

1

Prepare your reference image

Take a photo or use an existing illustration. Crop it to roughly square. Save as reference.jpg in your project folder.

This is what the model uses to match your face and character across every episode. Use the same reference image every time — never swap it mid-series.

2

Edit the prompt in generate-cover.py

Open generate-cover.py. Find POSITIVE_PROMPT and update:

  • The hook word (top-left, small)
  • The scene elements (what floats around the character)
  • The character's emotion/action
  • Your series name (bottom text)

See docs/prompts.md for the full prompt structure and tips.

3

Run and iterate on the seed

Set CONTROL_AFTER_GENERATE = "randomize" and run 3–5 times:

python3.11 generate-cover.py

When you find an output where the character looks right, note the seed value that was used and lock it: SEED = your_number, CONTROL_AFTER_GENERATE = "fixed".

That seed is your character. Lock it and never change it.

It took 20+ iterations to lock the character in ep-07.

Wrong face. Wrong hair. Wrong person every seed. The prompt structure in docs/prompts.md is the version that finally worked consistently. Start there.

Using the pipeline
Creating
your slides.

Each episode gets its own script — a copy of generate-episode.py with the slide content filled in. The script opens every slide as it generates so you can review in real time.

The three slide types:

Type When to use Function
Illustrated You have a ComfyUI background. Text overlays the scene at the bottom. make_illustrated()
Dark text Statement or insight — no background image needed. Dark with amber bars. make_dark()
Cream CTA End of episode engagement question. Cream background, navy layout. make_cta()
Onboarding Always the last slide. "Most AI content shows you the result..." make_onboarding()

The typography hierarchy — same on every slide:

Role Font Color Size guide
Headline Anton Regular White #F1F1F0 120–150px
Emphasis word Playfair Display Bold Italic Amber #D97706 96–110px
Body text Playfair Display Italic Gray #A0A0A0 32–44px
Footer Playfair Display Italic Gray 28px always
Text position

All illustrated slides use a fixed start_y = 1650 — same vertical position on every slide regardless of how many lines of text. This is what gives the series visual consistency when scrolling through.

From nine episodes of building this
What I
learned.
—

Lock your character early

Generate 5–10 covers with "randomize" seed before you commit to anything. The character drift is real — once you find a seed where it looks right, that seed is your series identity. Everything else can change. Not the seed.

—

Use the same reference image every episode

Never swap your reference image mid-series. It's the anchor for character consistency. Even a slightly different crop or brightness will cause drift.

—

Read the whole error — not just the first line

The most useful information is usually at the bottom of an error message. The "authentication method not allowed" issue took three hours because I kept reading the 200 status and missing the actual error in the response body.

—

The WebSocket only broadcasts the filename once

When a ComfyUI job completes, the output filename is broadcast exactly once via the "executed" event. If you miss it — the file is gone. The scripts in this repo capture it correctly. Don't restructure the WebSocket loop.

—

Describe output to Claude Code — not code

Don't say "write a PIL function that draws a rectangle at coordinates X". Say "I want a slide with a dark background, a big white headline in the top half, and an amber italic word below it." Claude Code handles the implementation.
